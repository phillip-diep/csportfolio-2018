How does the project work overall?

The project works by using the DOM to manipulate how the website appeared. Elements would be stored in JavaScript using .getElementById. They would then have their properties changed in many ways: css styling could be changed using .style, html itself could be changed using .innerHTML. The code also used .addEventListener to many buttons. This command would run functions when a certain event happened to the assigned object. I also used a if / else if statement to create input/output system.

What are some specifics of the code and how do those work?

In the code, .getElementById was used heavily. An example of it is the line: 

 
    var input = document.getElementById("input");

This line of code works by first creating a variable in JS, and then assinging it to the .getElementById. The .getElementById is writen always as document.getElementById("name"). It looks for anything with the same id as what is in the quotation marks and calls it to the JS code. If getElementById is not used, then the JavaScript would not be able to change anything in the html code. The elements need to be defined and called using .getElementById or an other similiar function.

In the code, .style was used to change the styles of many elements. An example of it is in the lines: 

 
    opening.style.opacity = "0";
    input.style.display = "flex";
    error.style.display = "block";

In these lines of code, there are many .style codes being used. In the first line, .style is paired with .opacity because that is what we are changing. When using .style, the specific css property being changed follows as style.NAME. The first line will change the opacity of "opening" to 0, change the display of "input" to flex, and change the display of "error" to block. This will overall make the opening slide fade away, make the input slide appear in its place, and it will create the error that will be shown if the thickness value is too big. These .style changes, with other ones, are how I got my different slides to tranistion from one to the other smoothly.

In the code, .innerHTML was used to change to html of the results. An example of it is in the lines: 

 
    side1.innerHTML = “<h4> 5 minutes </h4>”
    side2.innerHTML = “<h4> 3 minutes </h4>”

These 2 pieces of code are one result for the output. There are 2 innerHTMLs being changed: side1 and side2. There two variables are div elements, so when the innerHTML is changed, the results will be displayed in the divs in a <h4> style. I used .innerHTML because I needed to have different displays for different results, and if I had used .style, there would have been an uneccicary amount of created text: by using .innerHTML, I was able to cut down on the amount of text I had to actually create.

In the website, there are many buttons that have many functions. To have them cause events to happen, .addEventListener has to be used. An example of it is in the lines: 

 
    exitAlert.addEventListener("click", function() {
    error.style.top = "-37vw";
    });

This piece of code is a complete .addEventListener. There is a variable which the eventListener will be added to (exit.alert), a event that is being listened for ("click"), and a fucntion that is run when the event happens.

In the website, the output is determined using a system of if / else if statements. There are 4 different if / else if statements; each one is assigned to a different cooking value (rare, med. rare, medium, well done). In each if / else if statement, there are ranges. These ranges are written as a less than or equal to the biggest number in that range. For each range, there is a different output. An example of one of these ranges is:

 
    else if (cooking.value === "Rare" && meatSize <= 150) {
        side1.innerHTML = "<h4> 6 minutes </h4>";
        side2.innerHTML = "<h4> 4 minutes </h4>";
    }

This range is for the rare cooking choice. In this range, it is everything that is equal or less than 150, which equals 1.5 inches. The range before this range had a highest value of 125, which is equal to 1.25 inches. This makes this specific range apply to when the cooking value is rare and when the input for thickness is inbetween 1.25 inches and 1.5 inches.

What choices did you have to make and how did you go about making them?

Some choices I had to make was to create what inputs and outputs there would be. For my website, I had the idea to create a guide for how long to cook meat. This meant that I choose to use a 2 input system. This caused me to make multiple if / else if statements because there needed to bea first if about the first input and then a second if after that based on the second input. I also chose to use .style to hide my different slides because if I had used .innerHTML, the buttons and input wouldn't have worked. By making them hidden by using .style, I was able to still have it look how I wanted and also have the program work.

How has this project changed your connection and understanding of what it means to interact with something?

From this project, I have learned that interaction is being able to change an object and to be able to control when or how it changes. Through the us of JS DOM, I have created interactions from a small color change during a hover or to a large output from a click.
