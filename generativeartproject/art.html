How to:

Balloons:

For the balloons, I first had to draw one balloon. I used a ellipse, a triangle, an arc, and a bezier to create the balloon.

After creating one balloon, I made the balloon into a function, so that I could use many of them. The function looked like this: 


  function Balloon(xpos, ypos, r, g, b)

This function creates a balloon, but it also has the x position, the y position, and the value for rgb (color) as parameters. This allows the balloon to be customized.

Balloon Animation:

After creating the Balloon function, it needed to be used to create an animation where balloons would rise up and have random positions and colors. To do this, I first created an array. This array would then hold each individual balloon with their random position and color attached. To create the balloons that would be pushed into the array, I used a for loop. The loop looked like this: 


  for (var b = 0; b < 500; b++) { 
  balloons.push(new Balloon(random(0, 1000), random(height, 100000), random(255), random(255), random(255)));
  }

This loop will run until 499 balloons are created, and each balloon created has its position and rgb value randomly chosen. Each balloon created is automatically pushed into the array, where it is stored for usage later. The balloons y-position starts under the screen, as it is randomly assigned a y-value between the range of the bottom of the canvas to the y position of 100,000; this creates the effect of the balloons rising from out of the canvas.

To create the animation of the balloons rising up, I first made a draw function so that the animation moved. First, I added background(); so that the past frame would be cleared. Next, I had to draw all of the balloons. So I used a for loop to draw the balloons. The code looked like this: 


  for (var i = 0; i < balloons.length; i++) { 
    balloons[i].draw();
  }

This loop runs until all of the balloons are created; the amount of balloons stored in the balloons array determined this number. After, the balloons had to animated to rise up. Since I also wanted the speed of the balloon's rise to be changed by the position of the mouse, I used a if/if else statement. The statement looked like this: 


  if (mouseY < 150) { 
    balloons[i].ypos -= 10;
  }
  else if (mouseY > 350) {
    balloons[i].ypos -= 2.5;
  }
  else {
    balloons[i].ypos -= 5;
  }

This statement works by using the y position of the mouse - defined as mouseY.
If the mouse position is less than 150, which is roughly the top third of the canvas, than the balloons will rise at the quickest speed.
If the mouse position is less than 350, which is roughly the bottom third of the canvas, than the balloons will rise at the slowest speed.
And if the mouse is not in these two ranges, which is roughly the middle third of the canvas, than the balloons will rise at the middle speed.
The balloons rise because in each option, their y position is decreased by a different value: the higher the value, the quicker the speed.

Clouds:

To create the clouds, I first made 3 different clouds. They were made using a bunch of ellipses. Each cloud type had a name assigned: there was cloud1, cloud2, and cloud3. After the clouds were created into functions with parameters for x and y, they could be animated.

Animating the Clouds:

The clouds were animated inside the draw function. Each cloud was animated in the same way. The code animating for one of the clouds looked like this: 


  cloud1(cloud1x, 100);
  if (cloud1x < width) {
    cloud1x += 10;
  }
  else if (cloud1x >= width) {
    cloud1x = 0;
  }

The first line of code calls for a cloud (in this case, cloud1) and defines the x as cloud1x and the y as 100. For each cloud, the x was defined as the variable cloud(#)x. Each cloud had a different number, so that each cloud could move at different speeds. For the y values, a height for the cloud was given; each cloud had a different height. After the cloud was created, an if/else if statement was used to make sure that the cloud reset after it hit the edge of the canvas. The cloud would only move if cloud(#)x was less than the width of the canvas, and if cloud(#)x was greater than or equal to the width, than its value would return to 0, which resets the animation.

Party Mode!

Party mode is where the animations move faster, the clouds and background have flashing colors, and where the text Party! appears. This mode is activated when the mouse is clicked.

The first part of creating this mode is using the mouseClicked function to cause the mode to start only when the mouse is clicked. After that, another draw function is put underneath the function to create the animations. The function is very similar to the normal one, only with a few changes.

Faster Animations:

To create faster animations, any animations in the copied draw function had their rate increased. For example, the speed of the balloons when the mouse is in the middle third is 5px normally, but in party mode the speed is 50px.

Text:

To create the text Party!, I drew out the letters and then made them into a function. This allowed me to put the text anywhere I wanted.

Flashing Colors:

To create the flashing colors, I first had to create variables for the rgb values of the clouds, the background, and the text. After the variables were created, I could use the random function to change the value of the variables, which would make the rgb values of the clouds, background, and the text different every time.

Reflection:

While doing this project, I experimented with a lot of ways to make what I wanted happen. These experiments did not always turn out well or even work at all.

One of these missteps was when creating the looping animation for the clouds. The first time, I made the animation reset only when the x position was equal to the width of the canvas. This was a mistake because the animation would only reset if the x position was equal to the width of the canvas. That meant that the clouds could only travel at speeds that were factors of the width (which was 1000px). I fixed this issue by making the animation restart when the x position of a cloud was equal to or greater than the width of the canvas. This allowed the clouds to move at any speed.

Another misstep I made was when creating party mode. When I first created party mode, I just used the fill function with the 3 rgb values containing a random function in them to color in the clouds. This did not work however, because then the outlines of the clouds' ellipses were visible; they were still white. That meant that I had to create variables that would randomize a r,g, and b value that I could apply both the the fill and stroke functions, which would color both the cloud and its outlines.

Overall, after some experimentation and building upon the results, I was able to create my art.
